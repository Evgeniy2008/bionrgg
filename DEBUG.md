# Отладка проблемы "Профіль не знайдено"

## Шаги для диагностики:

### 1. Проверьте базу данных
Откройте в браузере: `http://ваш-домен/test-db.php`
- Должны увидеть таблицы users и users_info
- Проверьте, есть ли ваш пользователь в обеих таблицах

### 2. Проверьте URL
Откройте в браузере: `http://ваш-домен/test-url.php/ваш-username`
- Проверьте, правильно ли извлекается username из URL
- Проверьте, найден ли пользователь в базе данных

### 3. Проверьте логи сервера
Посмотрите в логах Apache/PHP на сообщения:
- "Searching for username: ..."
- "Found X rows for username: ..."

### 4. Проверьте .htaccess
Убедитесь, что файл .htaccess находится в корне проекта и содержит правила для перенаправления.

### 5. Проверьте права доступа
Убедитесь, что:
- База данных `bionrgg` существует
- Таблицы `users` и `users_info` созданы
- Пользователь `root` с паролем `root` имеет доступ к базе данных

## Возможные причины проблемы:

1. **Неправильный URL**: После регистрации перенаправление идет на `/username` вместо `profile.html?user=username`
2. **Проблемы с .htaccess**: Правила перенаправления не работают
3. **Проблемы с базой данных**: Пользователь не создался или создался в неправильной таблице
4. **Проблемы с кодировкой**: Символы в username не сохраняются правильно

## Решение:

1. Убедитесь, что после регистрации вы перенаправляетесь на `/username`
2. Проверьте, что .htaccess правильно перенаправляет `/username` на `profile.html`
3. Проверьте, что в базе данных есть записи в обеих таблицах
4. Проверьте логи сервера для отладочной информации
